# PostgreSQL Database Driver Implementation

This document describes the PostgreSQL database driver implementation for the mendelson AS2 server.

## Overview

The PostgreSQL driver manager has been implemented based on the HSQLDB logic, providing full support for PostgreSQL as the backend database with flexible configuration options.

## Implementation Details

### 1. Main Components

#### PostgreSQLConfig.java
Configuration loader with priority-based configuration reading:

**Priority Order (Highest to Lowest):**
1. Environment Variables (`POSTGRES_*`)
2. System Properties (`-Dpostgresql.*`)
3. Configuration File (`config/database-postgresql.properties`)
4. Default Values

**Features:**
- Automatic configuration loading from multiple sources
- Validation and type conversion
- Connection pool configuration
- SSL/TLS support
- Debug logging of active configuration

#### DBDriverManagerPostgreSQL.java
Main driver manager implementation with the following features:

- **Connection pooling**: Uses HikariCP for both CONFIG and RUNTIME databases
- **Flexible configuration**: Reads from PostgreSQLConfig for all settings
- **Database naming**: Configurable database names (default: `as2_db_config`, `as2_db_runtime`)
- **JDBC URL format**: `jdbc:postgresql://host:port/database_name`
- **Binary data handling**: Uses `setBytes()` and `getBytes()` for BYTEA columns
- **Singleton pattern**: Thread-safe singleton instance
- **Pool monitoring**: Provides connection pool statistics

#### DBServerPostgreSQL.java
Database server wrapper for PostgreSQL:

**Key Differences from HSQLDB:**
- PostgreSQL runs as an **external server process** (not embedded)
- No need to start/stop the database server itself
- Manages database initialization and schema updates
- Handles connection pool lifecycle
- Performs database version checking and migrations

**Features:**
- Database existence checking
- Automatic table creation from CREATE.sql scripts
- Version-based database migration support
- Transaction management for updates
- Connection pool setup and shutdown
- Database health monitoring

#### AbstractDBDriverManagerPostgreSQL.java
Base class providing PostgreSQL-specific database operations:

**Table Locking:**
- Exclusive locks: `LOCK TABLE ... IN EXCLUSIVE MODE`
- Insert/Update locks: `LOCK TABLE ... IN ROW EXCLUSIVE MODE`
- Delete locks: `LOCK TABLE ... IN ROW EXCLUSIVE MODE`

**Transaction Management:**
- Start: `BEGIN TRANSACTION ISOLATION LEVEL READ COMMITTED`
- Commit: `COMMIT`
- Rollback: `ROLLBACK`

**Data Type Handling:**
- Text: Uses `VARCHAR` type with standard JDBC methods
- Binary: Uses `BYTEA` type (via `setBytes()`/`getBytes()`)
- Objects: Stored as PostgreSQL objects

### 2. Database Schema

The SQL scripts have been converted to PostgreSQL syntax:

**config/CREATE.sql:**
- Changed `BLOB` → `BYTEA`
- Changed `INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY` → `SERIAL PRIMARY KEY`
- Added `keydata` table for keystore storage

**runtime/CREATE.sql:**
- Changed `OTHER` type → `TEXT` (for serialized objects)
- Changed auto-increment syntax to `SERIAL`
- Converted all table and index names to lowercase

## Configuration

The PostgreSQL driver supports multiple configuration methods with the following priority (highest to lowest):

1. **Environment Variables** (highest priority)
2. **System Properties** (via -D flags)
3. **Configuration File** (`config/database-postgresql.properties`)
4. **Default Values** (lowest priority)

### Method 1: Configuration File (Recommended)

Edit the `config/database-postgresql.properties` file:

```properties
# Database server settings
postgresql.host=localhost
postgresql.port=5432

# Database credentials
postgresql.user=as2user
postgresql.password=as2password

# Database names
postgresql.db.config=as2_db_config
postgresql.db.runtime=as2_db_runtime

# Connection pool settings (optional)
postgresql.pool.maximumPoolSize=10
postgresql.pool.minimumIdle=2
postgresql.pool.connectionTimeout=30000
postgresql.pool.idleTimeout=600000
postgresql.pool.maxLifetime=1800000

# SSL settings (optional)
postgresql.ssl.enabled=false
postgresql.ssl.mode=require
```

### Method 2: Environment Variables

Use environment variables for sensitive information or containerized deployments:

```bash
# Copy the example file
cp config/database-postgresql.env.example config/database-postgresql.env

# Edit the file with your settings
nano config/database-postgresql.env

# Source it before running the application
source config/database-postgresql.env
java -jar mendelson-as2.jar
```

**Available Environment Variables:**
- `POSTGRES_HOST` - Database server hostname
- `POSTGRES_PORT` - Database server port
- `POSTGRES_USER` - Database username
- `POSTGRES_PASSWORD` - Database password
- `POSTGRES_DB_CONFIG` - Config database name
- `POSTGRES_DB_RUNTIME` - Runtime database name
- `POSTGRES_POOL_MAX_SIZE` - Maximum pool size
- `POSTGRES_POOL_MIN_IDLE` - Minimum idle connections
- `POSTGRES_POOL_CONN_TIMEOUT` - Connection timeout (ms)
- `POSTGRES_POOL_IDLE_TIMEOUT` - Idle timeout (ms)
- `POSTGRES_POOL_MAX_LIFETIME` - Max connection lifetime (ms)
- `POSTGRES_SSL_ENABLED` - Enable SSL (true/false)
- `POSTGRES_SSL_MODE` - SSL mode (require/prefer/disable)

### Method 3: System Properties

Set system properties via JVM arguments:

```bash
-Dpostgresql.host=localhost      # Default: localhost
-Dpostgresql.port=5432           # Default: 5432
-Dpostgresql.user=as2user        # Default: as2user
-Dpostgresql.password=as2password # Default: as2password
-Dpostgresql.db.config=as2_db_config
-Dpostgresql.db.runtime=as2_db_runtime
```

### Example JVM Arguments

```bash
java -Dpostgresql.host=db.example.com \
     -Dpostgresql.port=5432 \
     -Dpostgresql.user=as2admin \
     -Dpostgresql.password=SecurePass123 \
     -jar mendelson-as2.jar
```

### Configuration Priority Example

```bash
# If all three are set:
export POSTGRES_HOST=prod-db.example.com        # Priority 1 (used)
-Dpostgresql.host=test-db.example.com          # Priority 2 (ignored)
# postgresql.host=localhost in properties file  # Priority 3 (ignored)
# Default: localhost                             # Priority 4 (ignored)

# Result: Connects to prod-db.example.com
```

## Database Setup

### Prerequisites

1. **PostgreSQL server** installed and running (version 12 or later recommended)
2. **PostgreSQL JDBC driver** - Already included in `pom.xml`
3. **Maven** - To build the project and download dependencies
4. **Network access** to PostgreSQL server

### Step 1: Build the Project

First, ensure the PostgreSQL JDBC driver is available:

```bash
# Download dependencies and build
mvn clean install

# Or if you just want to download dependencies
mvn dependency:resolve
```

The PostgreSQL JDBC driver (`postgresql-42.7.4.jar`) will be automatically downloaded to your local Maven repository and included in the build.

### Step 2: Quick Setup (Recommended)

Use the automated setup script:

```bash
# Make the script executable
chmod +x config/setup-postgresql.sh

# Run the setup script
./config/setup-postgresql.sh
```

The script will:
1. Prompt for database configuration
2. Create the AS2 user and databases
3. Generate the configuration file automatically
4. Set appropriate permissions

### Manual Database Creation

If you prefer manual setup:

```sql
-- Connect to PostgreSQL as superuser or user with CREATEDB privilege
psql -U postgres

-- Create user
CREATE USER as2user WITH PASSWORD 'as2password';

-- Create databases
CREATE DATABASE as2_db_config OWNER as2user;
CREATE DATABASE as2_db_runtime OWNER as2user;

-- Grant privileges
GRANT ALL PRIVILEGES ON DATABASE as2_db_config TO as2user;
GRANT ALL PRIVILEGES ON DATABASE as2_db_runtime TO as2user;
```

Then edit `config/database-postgresql.properties` with your settings.

### Docker Setup

Use Docker Compose for containerized deployment:

```bash
# Start PostgreSQL and AS2 server
docker-compose -f config/docker-compose-postgresql.yml up -d

# View logs
docker-compose -f config/docker-compose-postgresql.yml logs -f

# Stop services
docker-compose -f config/docker-compose-postgresql.yml down
```

The Docker setup includes:
- PostgreSQL 15 with automatic database initialization
- AS2 server configured to connect to PostgreSQL
- Persistent volumes for data storage
- Health checks for service dependencies

### Automatic Table Creation

Once the databases exist, the AS2 application will automatically:
1. Connect to each database
2. Check if the `version` table exists
3. If not, execute the CREATE.sql scripts to create all tables
4. Insert initial version information

## Connection Pooling

HikariCP is used for connection pooling with separate pools for:
- **CONFIG database**: Configuration, certificates, partners, keystores
- **RUNTIME database**: Messages, MDN, logs, statistics

### Default Pool Configuration

```properties
postgresql.pool.maximumPoolSize=10      # Max connections per pool
postgresql.pool.minimumIdle=2           # Min idle connections
postgresql.pool.connectionTimeout=30000  # 30 seconds
postgresql.pool.idleTimeout=600000      # 10 minutes
postgresql.pool.maxLifetime=1800000     # 30 minutes
```

### Tuning for Production

For high-load environments:

```properties
# High-throughput configuration
postgresql.pool.maximumPoolSize=50
postgresql.pool.minimumIdle=10
postgresql.pool.connectionTimeout=10000

# Or via environment variables
export POSTGRES_POOL_MAX_SIZE=50
export POSTGRES_POOL_MIN_IDLE=10
```

### Monitoring Connection Pools

## Key Differences from HSQLDB

| Feature | HSQLDB | PostgreSQL |
|---------|--------|------------|
| Connection | File-based or server mode | TCP/IP network connection |
| Configuration | Hardcoded or system properties | Multi-source config (env/props/file) |
| Binary data | JAVA_OBJECT type | BYTEA with standard JDBC |
| Auto-increment | IDENTITY columns | SERIAL type |
| Lock syntax | `LOCK TABLE x READ,x WRITE` | `LOCK TABLE x IN ... MODE` |
| Transaction | `START TRANSACTION` | `BEGIN TRANSACTION` |
| Object storage | Native Java serialization | TEXT or BYTEA |
| Database names | AS2_DB_CONFIG | as2_db_config (lowercase) |
| Pool config | Hardcoded | Fully configurable |

## Usage

### Switching from HSQLDB to PostgreSQL

In `AS2Server.java` or the main server configuration, change the database driver manager:

```java
// Old HSQLDB
IDBDriverManager dbDriverManager = DBDriverManagerHSQL.instance();

// New PostgreSQL
IDBDriverManager dbDriverManager = DBDriverManagerPostgreSQL.instance();
```

## Migration from HSQLDB

To migrate existing data from HSQLDB to PostgreSQL:

### Step 1: Export from HSQLDB

```sql
-- Use HSQLDB's SCRIPT command
SCRIPT 'export.sql';

-- Or export specific tables
SELECT * FROM partner;
```

### Step 2: Prepare PostgreSQL

```bash
# Run setup script
./config/setup-postgresql.sh
```

### Step 3: Import to PostgreSQL

```bash
# Import data (may need schema adjustments)
psql -U as2user -d as2_db_config -f export.sql

# Or use pg_restore for binary dumps
pg_restore -U as2user -d as2_db_config export.dump
```

### Step 4: Update Application

```bash
# Edit configuration to use PostgreSQL
nano config/database-postgresql.properties

# Or set environment variables
export POSTGRES_HOST=localhost
export POSTGRES_USER=as2user
export POSTGRES_PASSWORD=as2password
```

### Step 5: Test and Verify

```bash
# Start application
java -jar mendelson-as2.jar

# Verify data
psql -U as2user -d as2_db_config -c "SELECT COUNT(*) FROM partner;"
```

## Troubleshooting

### Connection Errors

**Error: Database does not exist**
- Ensure databases are created: `as2_db_config` and `as2_db_runtime`
- Run: `./config/setup-postgresql.sh` or create manually

**Error: Password authentication failed**
- Verify credentials in configuration
- Check `pg_hba.conf` for authentication method
- Test connection: `psql -h localhost -U as2user -d as2_db_config`

**Error: Connection refused**
- Check PostgreSQL is running: `pg_isready -h localhost -p 5432`
- Verify host and port in configuration
- Check firewall settings
- Ensure PostgreSQL accepts TCP/IP connections (check `postgresql.conf`)

### Configuration Not Loading

**Properties file not found:**
```
WARNING: Could not load config/database-postgresql.properties, using defaults
```
- Check file exists: `ls -l config/database-postgresql.properties`
- Verify file permissions are readable
- Use absolute path if needed

**Environment variables not working:**
```bash
# Verify variables are set
echo $POSTGRES_HOST
env | grep POSTGRES

# Make sure to export them
export POSTGRES_HOST=localhost  # Correct
POSTGRES_HOST=localhost         # Wrong (not exported)
```

### Table Lock Timeouts

If you experience lock timeout errors:
```sql
-- Increase lock timeout in PostgreSQL
ALTER DATABASE as2_db_config SET lock_timeout = '10s';
ALTER DATABASE as2_db_runtime SET lock_timeout = '10s';

-- Or globally in postgresql.conf
lock_timeout = 10s
```

### Performance Issues

For better performance with PostgreSQL:

1. **Create indexes** on frequently queried columns
   ```sql
   CREATE INDEX idx_messages_state ON messages(state);
   CREATE INDEX idx_messages_sender ON messages(senderid);
   ```

2. **Vacuum regularly** to maintain table statistics
   ```sql
   VACUUM ANALYZE;
   ```

3. **Adjust connection pool size** based on load
   ```properties
   postgresql.pool.maximumPoolSize=50
   postgresql.pool.minimumIdle=10
   ```

4. **Monitor query performance**
   ```sql
   -- Enable pg_stat_statements
   CREATE EXTENSION pg_stat_statements;

   -- View slow queries
   SELECT * FROM pg_stat_statements ORDER BY total_time DESC LIMIT 10;
   ```

5. **Tune PostgreSQL settings** in `postgresql.conf`
   ```
   shared_buffers = 256MB
   effective_cache_size = 1GB
   work_mem = 16MB
   maintenance_work_mem = 128MB
   ```

## Security Considerations

1. **Use strong passwords** for database users
   ```bash
   # Generate strong password
   openssl rand -base64 32
   ```

2. **Restrict network access** using `pg_hba.conf`
   ```
   # Allow only specific hosts
   host    as2_db_config   as2user   10.0.0.0/24   scram-sha-256
   host    as2_db_runtime  as2user   10.0.0.0/24   scram-sha-256
   ```

3. **Enable SSL/TLS** for encrypted connections
   ```properties
   postgresql.ssl.enabled=true
   postgresql.ssl.mode=require
   ```

4. **Never commit passwords to git**
   - Use environment variables for production
   - Use secrets management (Vault, AWS Secrets Manager, etc.)
   - Verify `.gitignore` excludes config files

5. **Regular backups** of both databases
   ```bash
   # Backup script
   pg_dump -U as2user as2_db_config > backup_config_$(date +%Y%m%d).sql
   pg_dump -U as2user as2_db_runtime > backup_runtime_$(date +%Y%m%d).sql
   ```

6. **Principle of least privilege** - grant only necessary permissions
   ```sql
   -- Create read-only user for reporting
   CREATE USER as2_readonly WITH PASSWORD 'readonly_pass';
   GRANT CONNECT ON DATABASE as2_db_runtime TO as2_readonly;
   GRANT SELECT ON ALL TABLES IN SCHEMA public TO as2_readonly;
   ```

## Monitoring and Maintenance

### Health Checks

```bash
# Check PostgreSQL status
pg_isready -h localhost -p 5432

# Check database sizes
psql -U as2user -d postgres -c "\l+"

# Check table sizes
psql -U as2user -d as2_db_runtime -c "
SELECT schemaname, tablename,
       pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS size
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY pg_total_relation_size(schemaname||'.'||tablename) DESC;"
```

### Performance Monitoring

```sql
-- Active connections
SELECT count(*) FROM pg_stat_activity
WHERE datname IN ('as2_db_config', 'as2_db_runtime');

-- Slow queries (requires pg_stat_statements)
SELECT query, calls, total_time, mean_time
FROM pg_stat_statements
WHERE query NOT LIKE '%pg_stat_statements%'
ORDER BY mean_time DESC
LIMIT 10;

-- Index usage
SELECT schemaname, tablename, indexname, idx_scan
FROM pg_stat_user_indexes
ORDER BY idx_scan;
```

### Backup and Recovery

```bash
# Full backup
pg_dump -U as2user -Fc as2_db_config > config_backup.dump
pg_dump -U as2user -Fc as2_db_runtime > runtime_backup.dump

# Restore
pg_restore -U as2user -d as2_db_config -c config_backup.dump
pg_restore -U as2user -d as2_db_runtime -c runtime_backup.dump

# Automated backup script
#!/bin/bash
BACKUP_DIR=/backups/as2
DATE=$(date +%Y%m%d_%H%M%S)
pg_dump -U as2user -Fc as2_db_config > $BACKUP_DIR/config_$DATE.dump
pg_dump -U as2user -Fc as2_db_runtime > $BACKUP_DIR/runtime_$DATE.dump
find $BACKUP_DIR -name "*.dump" -mtime +7 -delete  # Keep 7 days
```
